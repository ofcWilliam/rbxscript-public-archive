AK Admin Library

A lightweight admin UI library for Roblox. Build floating windows with tabs, toggles, sliders, dropdowns, player lists, and notifications — loaded in a single line.

URL https://absent.wtf/AKADMINLIB/LIB.lua
CoreGui Support Tab System Player List Draggable & Resizable Toast Notifications Touch & Mouse
01 — setup

Installation

Load the library with loadstring in a LocalScript or executor. It attempts to parent to CoreGui first, then falls back to PlayerGui if access is denied.

lua
local lb = loadstring(game:HttpGet("https://absent.wtf/AKADMINLIB/LIB.lua"))()
02 — example

Quick Start

Understanding how tabs work is the most important part of using this library correctly. Read the rules below before writing any code.

✖
Tab Rule — Read This First
Every element you add goes into whichever tab is currently active. The active tab is controlled entirely by win:switchTab(tab). You must call switchTab before adding elements to any tab — including the first one if you want to be explicit. Elements are never automatically distributed across tabs.

The correct pattern for building a tabbed window is always the same: create all tabs first, then loop through each tab by calling switchTab and adding its elements, then call switchTab one final time on whichever tab you want visible on open.

1 Call win:addTab() for every tab you need. Store each return value in a variable.
2 Call win:switchTab(tabVar) to make a tab active.
3 Add all elements that belong to that tab immediately after.
4 Repeat steps 2–3 for every other tab.
5 At the very end, call win:switchTab(firstTab) to set which tab opens by default.
lua — two tab example
local lb  = loadstring(game:HttpGet("https://absent.wtf/AKADMINLIB/LIB.lua"))()
local win = lb.new("My Script")

local mainTab     = win:addTab("Main")
local settingsTab = win:addTab("Settings")

win:switchTab(mainTab)
win:addSection("Combat")
win:addToggle("ESP", false, function(v) print(v) end)
win:addSlider("Speed", 16, 100, 16, function(v) print(v) end)

win:switchTab(settingsTab)
win:addSection("Preferences")
win:addToggle("Dark Mode", true, function(v) print(v) end)

win:switchTab(mainTab)
⚠ The final win:switchTab(mainTab) at the bottom is what makes the Main tab visible when the window opens. Without it, the last tab you added elements to would be shown instead.

If you have no tabs at all, elements are added directly to the window's scroll frame — no switchTab call is needed.

03 — window

lb.new()

Creates a new floating window. Duplicate windows with the same title are automatically destroyed before the new one is created.

NEW win = lb.new( title? ) ▾

Attempts to parent to CoreGui, falls back to PlayerGui. If a window with the same title already exists it is destroyed first, so calling lb.new() again is safe for re-execution.

Parameters
title optional string Shown in the title bar as AK ADMIN • title. Also used as the de-duplication key.
Returns — window object
win:addTab(name) — create a tab and return its ref
win:switchTab(tab) — set the active tab
win:minimize() — toggle collapsed state
win:resize() — recalculate window height
win:notify(...) — show a toast
win:onClose(fn) — register a close callback
win:addConnection(signal, fn) — managed connection
All add* element methods
⚠ RightShift toggles the window's visibility. This is hardcoded and cannot be changed from script.

addTab()

Registers a new tab in the navigation bar and returns a tab reference. The tab reference is required by switchTab() — always store it.

RET tab = win:addTab( name ) ▾

Creates the tab button and an empty content container. addTab does not make a tab active on its own — you must always call win:switchTab(tab) explicitly before adding elements to it. Call all addTab calls first, then handle each tab's content separately.

Parameters
name string Label shown in the tab button.
Returns — tab ref
An opaque reference. Pass it to win:switchTab(tab) to target that tab for element insertion.
✖ Never add elements immediately after addTab() without calling switchTab() first. The elements will be placed in whichever tab was active before, not the one you just created.
lua
local tabA = win:addTab("Main")
local tabB = win:addTab("Settings")

win:switchTab(tabA)
win:addButton("Action", function() end)

win:switchTab(tabB)
win:addToggle("Option", false, function() end)

win:switchTab(tabA)

switchTab()

Makes a tab active. Every add* call after this goes into that tab. This is the only way to control which tab receives elements.

FN win:switchTab( tab ) ▾

Activates the given tab both visually (highlights the tab button) and as the insertion target for new elements. There is no implicit switching — you are always responsible for calling this.

Parameters
tab tab ref The value returned by addTab(). Do not pass a string or index.
✦ Call switchTab at the very end of your setup script on the tab you want shown by default. This sets the visual state correctly when the window first appears.
lua — full pattern
local t1 = win:addTab("Main")
local t2 = win:addTab("Misc")
local t3 = win:addTab("Settings")

win:switchTab(t1)
win:addButton("Goes into t1", function() end)
win:addToggle("Also t1", false, function() end)

win:switchTab(t2)
win:addButton("Goes into t2", function() end)

win:switchTab(t3)
win:addToggle("Goes into t3", true, function() end)

win:switchTab(t1)

minimize()

Toggles the window between its collapsed (title bar only) and expanded states. The window also has a built-in — button that calls this.

FN win:minimize() ▾

No parameters. Collapses the window to show only the title bar. Calling again restores the full window. Has a short debounce to prevent rapid toggling.

lua
win:minimize()

onClose()

Registers a callback that fires when the window's X button is pressed. All managed connections are disconnected and the ScreenGui is destroyed after callbacks run.

FN win:onClose( callback ) ▾
Parameters
callback function Called before the GUI is destroyed. Use this to clean up any state your script maintains.
lua
win:onClose(function()
    print("Window closed")
end)
04 — elements

addButton()

Adds a clickable button to the active tab.

FN win:addButton( name, callback ) ▾
Parameters
name string Label on the button. Supports text wrapping.
callback optional function Called on click.
lua
win:addButton("Teleport to Spawn", function()
    game.Players.LocalPlayer.Character.HumanoidRootPart.CFrame = CFrame.new(0, 5, 0)
end)

addToggle()

Adds an animated pill-style on/off toggle.

RET ctrl = win:addToggle( name, default, callback ) ▾
Parameters
name string Label shown to the left of the toggle.
default optional boolean Initial state. Defaults to false.
callback optional function(v: boolean) Fires on every state change with the new boolean value.
Returns — ctrl object
ctrl:set(bool) — set state, fires callback
ctrl:get() — returns current boolean
ctrl.Value — readable/writable property, does NOT fire callback
lua
local esp = win:addToggle("ESP", false, function(v)
    print("ESP:", v)
end)

esp:set(true)
print(esp:get())

addSlider()

Adds a draggable slider with a fill bar and live value readout. Works with both mouse and touch.

RET ctrl = win:addSlider( name, min, max, default, step, callback ) ▾
Parameters
name string Label shown above the slider track.
min number Minimum value. Must be strictly less than max.
max number Maximum value.
default optional number Starting value. Clamped to [min, max]. Defaults to min.
step optional number Snap increment. Defaults to 1. Pass a value like 5 to snap in increments of 5, or 0.1 for decimal steps. If omitted you can pass the callback directly in this position.
callback optional function(v: number) Fires continuously while dragging with the current snapped value.
Returns — ctrl object
ctrl:set(num) — set value programmatically, clamped and snapped, does not fire callback
ctrl:get() — returns current value
⚠ If min >= max the slider is silently skipped and a no-op object is returned. If step is omitted, the callback can be passed as the 5th argument directly.
lua
local spd = win:addSlider("Walk Speed", 16, 200, 16, function(v)
    game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = v
end)

local vol = win:addSlider("Volume", 0, 100, 50, 5, function(v)
    print(v)
end)

addTextBox()

Adds a labeled text input field. The callback fires only when the user presses Enter — not on every keystroke.

RET ctrl = win:addTextBox( name, placeholder, maxLength?, callback ) ▾
Parameters
name string Label shown above the input.
placeholder optional string Ghost text shown when the field is empty.
maxLength optional number Maximum character count. Defaults to 200. If omitted, the callback can be passed in this position directly.
callback optional function(v: string) Fires on FocusLost only when Enter was pressed. Not called on blur alone.
Returns — ctrl object
ctrl:get() — returns current text
ctrl:set(str) — programmatically set text
ctrl:clear() — clears the field
lua
local box = win:addTextBox("Target Player", "Enter username...", function(v)
    print("Submitted:", v)
    box:clear()
end)

addDropdown()

Adds a dropdown with a floating option panel. Options can be replaced at runtime.

RET ctrl = win:addDropdown( name, options, default, callback ) ▾
Parameters
name string Label to the left of the dropdown button.
options string[] Array of option strings. Must not be empty.
default optional string Initially selected value. Defaults to options[1].
callback optional function(v: string) Fires when the user selects an option.
Returns — ctrl object
ctrl:get() — returns currently selected string
ctrl:set(str) — programmatically select an option
ctrl:refresh(newOptions) — replace the option list entirely
ctrl:destroy() — remove the dropdown and clean up its connections
lua
local dd = win:addDropdown("Team", {"Red", "Blue", "Green"}, "Red", function(v)
    print("Picked:", v)
end)

local names = {}
for _, p in pairs(game.Players:GetPlayers()) do
    table.insert(names, p.Name)
end
dd:refresh(names)

addKeybind()

Adds a keybind row. Clicking enters listening mode — the next key pressed becomes the binding. Clicking again while listening clears the binding.

RET ctrl = win:addKeybind( name, default, callback ) ▾
Parameters
name string Label. Also used as the internal registry key — must be unique within the window.
default optional Enum.KeyCode Initial binding. Pass nil for unbound.
callback optional function Called every time the bound key is pressed. Not called during listening mode.
Returns — ctrl object
ctrl:get() — returns current Enum.KeyCode or nil
ctrl:set(keyCode) — assign a key programmatically
⚠ Keybind names must be unique per window. Two keybinds sharing the same name will overwrite each other in the internal registry.
lua
win:addKeybind("Toggle Fly", Enum.KeyCode.F, function()
    print("Fly toggled")
end)

addHoldButton()

Adds a hold-to-activate element. It becomes active while the user holds the mouse button down on it or holds the bound key. Releasing immediately deactivates it. A progress bar fills over the configured duration while held.

RET ctrl = win:addHoldButton( name, keyCode, holdDuration, onActive, onInactive ) ▾

Unlike a toggle, this element is only active for as long as the user physically holds the input. The moment the mouse button or key is released, onInactive fires and the bar resets. Both mouse and keyboard inputs work simultaneously — releasing either one while the other is still held keeps it active.

Parameters
name string Label shown on the element.
keyCode optional Enum.KeyCode Optional keyboard shortcut that also triggers hold. Pass nil for mouse-only.
holdDuration optional number How many seconds the progress bar takes to fill completely. Defaults to 1. This is cosmetic only — the element activates on press, not after the bar fills.
onActive optional function Fires once when the hold begins.
onInactive optional function Fires once when the hold ends (input released).
Returns — ctrl object
ctrl.isActive() — returns true while being held
ctrl.setActive(bool) — force active/inactive state and update the visual, does not fire callbacks
lua
win:addHoldButton("Sprint", Enum.KeyCode.LeftShift, 2,
    function()
        game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = 50
    end,
    function()
        game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = 16
    end
)

addPlayerList()

Adds a live player list with avatar thumbnails. Supports two distinct modes depending on the type of the first argument.

RET ctrl = win:addPlayerList( labelOrCallback, callback? ) ▾

The function behaves differently based on whether the first argument is a string or a function.

Action Mode — first argument is a string
label string Text shown on a button beside each player card (e.g. "Kick", "Teleport").
callback function(player) Fires with the Player instance when their action button is clicked.
Select Mode — first argument is a function
callback function(player, selected) Clicking a card toggles its selected state. Fires with the Player and a boolean. Only one player can be selected at a time.
Returns — ctrl object
ctrl.refresh() — rebuild the list from current server players
ctrl.getSelected() — returns selected Player or nil (select mode only)
✦ Avatars are loaded asynchronously so the list renders immediately and portraits fill in as they arrive.
lua — action mode
win:addPlayerList("Teleport", function(player)
    local char = player.Character
    if char then
        game.Players.LocalPlayer.Character.HumanoidRootPart.CFrame = char.HumanoidRootPart.CFrame
    end
end)
lua — select mode
local list = win:addPlayerList(function(player, selected)
    print(player.Name, selected)
end)

print(list.getSelected())
05 — layout

addSection()

Inserts a horizontal rule with a centered text label for grouping elements visually.

FN win:addSection( name ) ▾
Parameters
name string Text centered in the divider.
lua
win:addSection("Movement")
win:addToggle("Fly", false, function() end)
win:addSection("Visual")
win:addToggle("ESP", false, function() end)

addSeparator()

Inserts a thin horizontal line with no label — a minimal visual break between elements.

FN win:addSeparator() ▾

No parameters. Renders a 10px tall row containing a 1px separator line.

lua
win:addButton("Action A", function() end)
win:addSeparator()
win:addButton("Action B", function() end)

addLabel()

Adds a read-only text row. Supports dynamic updates.

RET ctrl = win:addLabel( text ) ▾
Parameters
text string Initial text. Wraps automatically.
Returns — ctrl object
ctrl:get() — returns current text
ctrl:set(str) — update displayed text
lua
local lbl = win:addLabel("v1.0.0 loaded.")
lbl:set("Connected to server.")

addStatus()

Adds a two-column row with a fixed label on the left and a dynamically updatable value on the right.

RET ctrl = win:addStatus( name, initialValue? ) ▾
Parameters
name string Static label on the left.
initialValue optional string Starting value text. Defaults to "—".
Returns — ctrl object
ctrl:get() — returns current value string
ctrl:set(str, color?) — update value text and optionally its Color3
lua
local ping = win:addStatus("Ping", "—")

task.spawn(function()
    while true do
        local ms  = math.round(game:GetService("Stats").Network.ServerStatsItem["Data Ping"]:GetValue())
        local col = ms < 100 and Color3.fromRGB(80,200,120) or Color3.fromRGB(224,92,92)
        ping:set(ms .. "ms", col)
        task.wait(1)
    end
end)
06 — misc

notify()

Shows a toast notification that slides in from the right edge of the screen. Multiple notifications stack and reflow automatically.

FN win:notify( title, body, imageId?, persistent? ) ▾
Parameters
title string Bold primary text.
body optional string Smaller text below the title.
imageId optional number Roblox asset ID as a plain number — no rbxassetid:// prefix. Renders a 38×38 icon.
persistent optional boolean If true, the notification never auto-dismisses. Defaults to false — 6 second lifetime.
✦ Notifications live in a separate AK_NOTIF_GUI ScreenGui and are independent of the main window. A click sound plays on each appearance.
lua
win:notify("Loaded", "Script is ready.")
win:notify("Update", "v2.0 available", 7733960981)
win:notify("Warning", "Low FPS", nil, true)

addConnection()

Connects a signal and registers it with the window so it is automatically disconnected when the window is closed.

RET conn = win:addConnection( signal, callback ) ▾

Use this instead of a bare signal:Connect(fn) for any connections your script makes inside the window's lifecycle. When the X button is pressed, all connections registered this way are disconnected automatically.

Parameters
signal RBXScriptSignal Any Roblox signal (e.g. RunService.Heartbeat, Players.PlayerAdded).
callback function The handler to connect.
Returns
RBXScriptConnection — same as :Connect() returns
lua
win:addConnection(game:GetService("RunService").Heartbeat, function(dt)
    print(dt)
end)
